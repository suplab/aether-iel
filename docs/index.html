Intelligence Maturity Model

5 Levels of Intelligence Maturity

The IMM gives teams a shared language for where they are and a clear path to where they need to be. Each level has explicit entry criteria and advancement gates.

L1
Prompt Experiments
One-off LLM calls, no memory, no agents. Output is ephemeral and non-reproducible.
No AIEL required
L2
AI Applications
RAG or fine-tuned models with persistent knowledge. Phases 1–3 apply.
Phases 1 – 3
L3
Agent Systems
Multi-agent orchestration, tool use, feedback loops. Confidence gate is mandatory.
Phases 1 – 7
L4
Cognitive Platforms
Persistent memory, identity, emotional context, full GDPR compliance.
Phases 1 – 10
L5
Distributed Intelligence
Federated memory, collective learning, autonomous evolution across agent meshes.
All 12 Phases
Lifecycle Artifacts

Templates & Deliverables

Every phase produces a primary artifact. These templates enforce consistent structure across all intelligence systems built on the Aether ecosystem.

Phase 1 · Strategy
Intelligence Brief
Defines why the system exists, what decisions it makes, authority boundaries, and governance requirements. Required before any code is written.
Phase 2 · Cognitive Architecture
Cognitive Design
Documents reasoning strategies, memory model, knowledge sources, agent topology, and LLM selection rationale.
Phase 5 · Agent Engineering
Agent Contract
Specifies agent identity, capabilities, tool access, confidence gate, decision hierarchy, failure behaviour, and governance sign-off.
Phase 7 · Evaluation
Evaluation Report
Documents accuracy, latency, confidence distribution, failure modes, and production readiness decision with sign-off.
Maturity Assessment
IMM Assessment
Current level determination across 5 dimensions: memory, agents, governance, learning, and evolution. Produces advancement roadmap.
Architecture
ADR — Architecture Decision Record
Captures every significant methodology or design decision with context, options considered, decision made, and consequences.
Golden Rules

8 Non-Negotiable Principles

These rules are hard constraints, not guidelines. They apply to every intelligence system built in the Aether ecosystem.

Rule 01
Documentation First
No implementation begins without a completed Intelligence Brief (Phase 1 artifact). Code without a brief is refused at review.
Rule 02
Decisions in ADRs
Every significant methodology or architecture decision is captured in an Architecture Decision Record with context and consequences.
Rule 03
Brief Before Code
Phase 1 artifact must exist and be reviewed before Phase 6 begins. No exceptions.
Rule 04
Confidence Gate at 0.8
Agent decisions with confidence < 0.8 always route to human review. autoEnforced=false. This gate cannot be disabled without governance approval.
Rule 05
Memory is First-Class
Memory schema and retention policy are Phase 4 deliverables — designed explicitly, not discovered during implementation.
Rule 06
Explicit Agent Authority
Every agent has a signed Agent Contract that defines the exact boundary of its authority. Agents cannot self-escalate.
Rule 07
Evaluation is Continuous
Phase 7 Evaluation is not a final gate — it repeats on every evolution cycle. Regression detection is automated.
Rule 08
Governance at Phase 1
Governance requirements are captured in the Intelligence Brief. Governance added later is governance that failed.
